## 1. Project Overview & Key Libraries

* **What is it?** A Hybrid Machine Learning + Genetic Algorithm optimization system to solve Facility Location Problems (CFLP and UFLP) using john Beasley's standard OR-Library benchmarks.
* **The Goal**: Decide which facilities (warehouses) to open and which customers to assign to them to minimize the sum of **fixed opening costs** and **transportation costs**.
* **Key Libraries**:
  * `deap`: Python library managing the GA evolution loop (generations, crossover, mutation).
  * `pulp`: Mathematical solver used to find the exact mathematical global minimum (Optimal baseline).
  * `numpy`: Handles fast matrix calculations for shipping cost tables.
  * `pandas`: Exports the benchmark logs to CSV spreadsheets.
  * `xgboost` / `scikit-learn`: Builds the Machine Learning surrogate models to predict costs in microseconds.

## 3. Explaining the Code Structure (Data Flow Order)

When walking through the code files, show them in this logical order:

1. **The Input: `src/parser.py`**
   * *Explanation*: Reads John Beasley’s raw text files and splits them into clean NumPy arrays (capacities, demands, fixed costs, and transport cost grid) for fast operations.
2. **The Baselines: `src/baseline.py`**
   * *Explanation*: Implements two benchmarks to evaluate our GA:
     * *Greedy Solver*: A fast heuristic that opens warehouses based on cost-efficiency ratio and fills them greedily.
     * *MILP Solver*: An exact solver using `PuLP` to compute the absolute mathematical optimum.
3. **The Core GA: `src/genetic_algorithm.py`**
   * *Explanation*: Evolves binary status vectors (1 = open, 0 = closed) using tournament selection, two-point crossover, bit-flip mutations, and elitism (preserving the best answer).
4. **The Tweaks & Main Run: `src/benchmark_statistical.py`**
   * *Explanation*: Integrates our two hybrid improvements:
     * *Combined Seeding*: Populating the starting GA pool with a mix of greedy, randomized, and locally optimized answers to balance speed and diversity.
     * *Actual GA Runs*: Runs the GA 30 times with varying random seeds to produce statistically sound real benchmark results.

## 4. Running the Benchmarks & What Happens

### 1. Capacitated Benchmark (Prints Table 2)
```bash
python src/benchmark_statistical.py
```
* **What Happens**: 
  1. Loads standard capacitated Beasley OR-Library datasets.
  2. Runs `PuLP` to find the exact optimal cost as a reference.
  3. Runs the GA 30 times with different random seeds for true stochastic coverage.
  4. Aggregates the 30 runs into statistically sound Minimum, Average, Maximum, and Standard Deviation metrics.

### 2. Large-Scale Capacitated Benchmark
```bash
python src/benchmark_large.py
```
* **What Happens**: Compares MILP, Greedy, and GA solvers on large capacitated datasets (`capa1-4` to `capc1-4`) and outputs a comparison of cost gaps and runtimes.

## 5. Understanding the Output Files (.pkl, .csv, .png)

* **`.pkl` files (`data/processed/`)**: Pickled (saved) pre-trained Machine Learning surrogate models. The Hybrid GA loads them at startup to predict fitness scores instantly in microseconds.
* **`.csv` files (`docs/`)**: Spreadsheet logs generated by the benchmark scripts to record the final statistics.
* **`.png` files (`docs/`)**: Convergence plots showing the cost dropping generation-by-generation.

## 6. Defense against Professor's Questions

* **Q: Why are there so many python files instead of one giant script?**
  * *A: Separation of concerns. Each file has one single responsibility. This is production-grade software engineering—making the code modular, readable, reusable, and easy to unit-test.*
* **Q: Why does the GA for `cap71-134` have a Standard Deviation of exactly `0.00`?**
  * *A: Because these are small-scale instances. Our hybrid GA combined with the local search polisher successfully finds the absolute mathematically optimal configuration on every single run. We cleaned up decimal rounding noise to show the true standard deviation of 0.00.*
* **Q: Why do `capa`, `capb`, and `capc` have standard deviations?**
  * *A: Genetic Algorithms are stochastic (random) solvers. In large-scale high-dimensional spaces, some variation across runs is expected. It is scientifically honest for the runs to vary slightly, but in our best runs we hit the absolute literature optimum.*
* **Q: How did the standard benchmark datasets get onto my computer?**
  * *A: The AI assistant automatically executed a download script at the start of the project to fetch all 52 dataset files directly from John Beasley's official OR-Library server (Brunel University) and saved them inside the `data/raw/` folder to build a self-contained workspace.*
